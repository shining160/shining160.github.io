Spark Core

一.什么是Spark？特点？

官网：spark.apache.org，是一个大规模的分布式数据处理引擎
类似MapReduce，Spark只有数据计算，不负责存储，所以不能完全取代Hadoop
特点：快、易用、通用、兼容
为什么学习Spark? 基于内存，计算速度快。
MapReduce 2.x的核心是Shuffle，但会产生大量IO操作，影响性能。而Spark基于内存，有更快的处理速度。

3.Spark HA有两种方式：

回顾一下HA：
（）HDFS、Yarn、HBase、Storm、Spark
（）单点故障的问题
（*）解决方案：HA（High Availability）

（1）基于文件目录的单点恢复（测试使用，不用于生产环境）
（*）本质：还是只有一个主节点master

恢复目录 —> 保存的是集群的状态和任务的信息
重新启动Master，会从恢复目录下读取状态信息

配置方式：
在spark-env.sh里对SPARK_DAEMON_JAVA_OPTS里设置：
spark.deploy.recoveryMode：设置为FILESYSTEM开启单点恢复功能，默认值：NONE
spark.deploy.recoveryDirectory：Spark保存恢复状态的目录
执行命令：

添加 export SPARK_DAEMON_JAVA_OPTS=”-Dspark.deploy.recoveryMode=FILESYSTEM -Dspark.deploy.recoveryDirectory=/root/training/spark-2.1.0-bin-hadoop2.7/recovery”

测试：执行一个Application：启动spark-shell命令行工具（作为一个独立的Application运行在集群上），查看恢复目录，可以看到多了一个app目录。

（2）基于Zookeeper的Standby的Master
（）复习Zookeeper:
（）相当于是一个“数据库”
（*）数据同步、选举功能、分布式锁‘

添加 export SPARK_DAEMON_JAVA_OPS=”-Dspark.deploy.recoveryMode=ZOOKEEPER -Dspark.deploy.zookeeper.url=node1:2181,node2:2181,node3:2181 -Dspark.deploy.zookeeper.dir=/spark”

另外，注释掉文件恢复的配置内容：
/ #export SPARK_MASTER_HOST=node1
/ #export SPARK_MASTER_PORT=7077

三、执行Spark的任务：两个工具

1.spark-submit：用于提交spark的任务（就是一个jar包）

举例：examples/jars/spark-examples_2.11-2.1.0.jar
应用：蒙特卡罗求PI（圆周率）

2.spark-shell：相当于REPL，命令行工具，作为一个独立的Application运行

两种运行模式：

（1）本地模式：不需要连接到spark集群，在本地直接运行（就像Eclipse），用于开发和测试
命令：bin/spark-shell
日志：Spark context availavle as ‘sc’ (master=local[*], app id=local-15009234552)

（2）集群模式：需要连接到集群
命令：bin/spark-shell –master spark://node1:7077
日志：Spark context available as ‘sc’ (master = spark://node1:7077, app id=app-2018102549890034-001)
在spark-shell中开发程序wordcount：
（）处理本地文件 直接打印结果
sc.textFile(“/root/temp/input/data.txt”).flatMap(.split(“ “)).map((,1)).reduceByKey(_+_).collect
result：Array[(String, Int)] = Array((is,1),(love,2),(capital,1),(Beijing,2),(China,2),(I,2),(of,1),(the,1))
（）处理HDFS的文件：输出到HDFS
sc.textFile(“hdfs://node1:9000/input/data.txt”).flatMap(.split(“ “)).map((,1)).reduceByKey(_+).saveAsTextFile(“hdfs://node1:9000/output/result”)
（*）单步运行wordcount —> 知识点：RDD
val rdd1 = sc.textFile(“/root/temp/input/data.txt”)
val rdd2 = rdd1.flatMap(.split(“ “))
val rdd3 = rdd2.map((,1))
val rdd4 = rdd3.reduceByKey(+_)
rdd4.collect()

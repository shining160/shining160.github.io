Git使用教程

1.什么是git？

git是一个分布式版本控制软件，与CVS、Subversion一类的集中式版本控制工具不同，它采用分布式版本库的做法，不需要服务器端软件，就可以运作版本控制。git不等于GitHub，git是一种技术，GitHub是一种使用了该技术提供服务的平台。

2.实现原理

git本身关心文件的整体性是否有改变，git拒绝保持每个文件的版本修订关系，因此查看一个文件的历史需要遍历各个history快照；git隐式处理文件更名，即同名文件默认为其前身，如果没有同名文件则在前一个版本中搜索具有相似内容的文件。
git更像一个文件系统，直接在本机上获取数据，不必连线到主机端获取数据。每个开发者都可有全部开发历史的本地副本，changes从这种本地repository复制给其他开发者。这些changes作为新增的开发分支被导入，可以与本地开发合并。
分支是非常轻量级的，一个分支仅是对一个commit的引用。

3.库目录

hooks：存储钩子的文件夹
logs：存储日志的文件夹
refs：存储指向各个分支的指针（SHA-1标识）文件
objects：存放git对象
config：存放各种设置文档
HEAD：指向当前所在分支的指针文件路径，一般指向refs下的某文件

4.安装配置

4.1 下载安装包

Git-2.13.3-64-bit.zip

4.2 git的安装配置（只有初次使用需要配置）

用户名和邮件
git config –global user.name “Janson”
git config –global user.email “mr_Janson@163.com“

4.3 最基础的本地git操作

git init #初始化为本地仓库
git add . #每次更新都要add，将文件添加到本地缓冲区
git commit -m “xxxx” #添加到本地仓库

4.4 生成秘钥和公钥（只需要生成一次）

ssh-keygen -t rsa
如果不清楚就一路按回车，大概三四次
程序会提示输入秘钥的文件名，一路按回车即可。然后要求输入一个密码，将来在使用秘钥的时候需要提供这个密码。可以输入，也可以不输入直接回车（无论是否输入，都会要求确认一次）。确认完毕，程序将生成一对秘钥存放在以下文件。
秘钥分成两个文件，存放在C://user/username/.ssh这个隐藏目录下，一个私钥（id_rsa）、一个公钥（id_rsa.pub）。

4.5 注册代码托管平台账号

最出名的是GitHub，纯英文，在国外，速度略慢
coding.net 国内速度不错
csdn
开源中国

4.5 添加公钥

找到setting，给key填一个名字，再将本地id_rsa.pub文件内的内容粘贴过去。
创建项目命名为Janson_demo，则系统创建的两个访问地址为：
HTTPS https://git.coding.net/Janson/Janson_demo.git
SSH git@git.coding.net:Janson/Janson_demo.git

在命令行中创建Git仓库并同步到远程仓库：
mkdir Janson_demo
cd Janson_demo
git init
echo “# Janson_demo” >> README.md
git add README.md
git commit -m “first commit”
git remote add orgin git@git.coding.net:Janson/Janson_demo.git
git push -u orgin master

4.6 把本地仓库和远程仓库关联

git remote add orgin git@git.coding.net:Janson/Janson_demo.git
命令讲解：
orgin 是我们给远程仓库取的名称，可以随意，用来区分多个远程仓库。

4.7 把本地代码推送到远程仓库

git push orgin master
强制覆盖远程仓库：
giy push orgin master -f

4.8 远程仓库和git的通俗理解

git其实主要是在本地操作，所谓commit，回滚都是指本地开发。push到远程仓库后根本不能回滚。git主要是用来做版本控制的，托管平台是git的云服务器（当然自己也可以架设一台服务器，但是有现成的就不那么麻烦了）。git本地操作完后上传到GitHub，或者把GitHub上的东西更新到本地，git相当于百度云客户端，托管后台相当于百度云后台。

4.9 add commit push的区别

【（本地工作目录）—add—>（缓冲区）—commit—>（本地的git仓库）====push====>【托管平台】
